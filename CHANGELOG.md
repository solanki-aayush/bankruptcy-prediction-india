# Changelog

All notable changes to this repository are documented here.

---

## [1.0.0] — Initial Public Release

### Repository Restructuring

**What changed:** Reorganised from a single flat notebook into an industry-standard
project layout with separate `src/`, `notebooks/`, `data/`, `results/`, and `docs/`
directories.

**Why:** A flat layout with a single notebook is appropriate for experimentation but
makes it hard for others to navigate, reproduce, or extend the work. The new structure
separates concerns: raw data lives in one place, source modules in another, and
outputs are clearly distinguished from inputs.

**Impact:** Any recruiter or collaborator can immediately orient themselves — they know
where to look for models, data, and results without reading every file.

---

### Code Refactoring (`src/` modules)

**What changed:** The monolithic notebook was decomposed into four focused Python
modules:

| Module | Responsibility |
|---|---|
| `data_preprocessing.py` | Loading, feature selection, train/test split, scaling |
| `feature_engineering.py` | PCA dimensionality reduction and variance analysis |
| `train.py` | Model training loops and serialisation |
| `evaluate.py` | Metrics computation and all visualisations |
| `utils.py` | Shared helpers (plot style, model I/O, logging) |

**Why:** Monolithic notebooks are hard to test, import, or reuse. Functions in `.py`
files can be imported by other notebooks or scripts, enabling composability. This also
makes the logic unit-testable in future.

**Specific improvements:**

- `GradientBoostingClassifier` retained (this was already XGBoost-equivalent in the
  original; `xgboost.XGBClassifier` is also listed as an option in comments)
- Variable names standardised (`X_train_scaled` instead of `X_train` post-transform,
  to clarify state)
- Constants (`RANDOM_STATE`, `TEST_SIZE`, `CORRELATION_THRESHOLD`) extracted to
  module-level for easy configuration
- All debug `print` statements replaced by informative logging inside functions
- Duplicate import blocks removed (seaborn was imported twice in the original)

---

### Notebook Improvements

**What changed:** Two clean, well-documented notebooks replace the original single
notebook. All intermediate cell outputs are cleared.

- `01_eda_and_preprocessing.ipynb` — EDA, class distribution, correlation analysis,
  preprocessing
- `02_model_training_and_evaluation.ipynb` — Training, evaluation, comparison

**Why:** Splitting EDA from modelling is a standard practice. It keeps notebooks
shorter, more focused, and easier to present. Cleared outputs prevent stale results
from being committed and reduce file size.

**Markdown improvements:** Every major section has a prose explanation of *why* the
step is being done, not just *what*. Tables and markdown headers make the notebooks
readable as standalone documents.

---

### README

**What changed:** Created a comprehensive `README.md` from scratch. The original
project had no README.

**Why:** A README is the first thing any visitor sees. Without one, even a technically
strong project will be ignored. The new README covers the business problem, dataset,
methodology (with a Mermaid workflow diagram), results table, setup instructions,
limitations, and future work — everything a recruiter or interviewer needs to evaluate
the project without running a single line of code.

---

### Reproducibility

**What changed:** Added `requirements.txt` and `environment.yml`.

**Why:** Without pinned dependencies, running someone else's notebook is a gamble.
These files allow anyone to recreate the exact Python environment in minutes with
`pip install -r requirements.txt` or `conda env create -f environment.yml`.

---

### GitHub Best Practices

**What changed:** Added `.gitignore`, `LICENSE` (MIT), and this `CHANGELOG.md`.

**Why:**
- `.gitignore` prevents accidental commits of large data files, model binaries, and
  notebook checkpoints — common mistakes that bloat repositories
- `LICENSE` is required for any open-source project; without it, others technically
  cannot legally use the code even if it is publicly visible
- `CHANGELOG.md` demonstrates professional software engineering habits

---

### Visualisations

**What changed:** All visualisations are now generated by functions in `evaluate.py`
and saved to `results/figures/` as 150 dpi PNG files. A consistent colour palette and
matplotlib style are applied throughout.

**Why:** Reproducible, saved figures can be dropped directly into presentations or
papers. A consistent visual style makes the project look polished. In the original
notebook, plots were rendered inline but not persisted.

**Figures added / improved:**

- Class distribution (bar + pie)
- Full correlation heatmap (63 features)
- Selected feature correlation heatmap (28 features)
- PCA variance bar chart with cumulative line
- Confusion matrices (per model)
- ROC curve comparison (full and PCA feature sets)
- Feature importance bar chart (Random Forest top 20)
- Grouped model comparison bar chart

---

### Data Handling

**What changed:** Added a `.gitignore` rule to exclude raw data and processed splits
from version control. A clear note in the README explains how to obtain the data.

**Why:** The raw dataset contains ProwessIQ data which is proprietary. Committing it
would violate licensing terms. The `.gitignore` rule prevents accidental exposure.
Processed splits (`.pkl` files) are derivatives of the raw data and similarly excluded.

---

### What Was NOT Changed

- **Methodology** — no changes to the models, hyperparameters, evaluation metrics, or
  the train/test split ratio. All results remain faithful to the published paper.
- **Feature selection logic** — the |r| > 0.5 multicollinearity threshold is preserved.
- **Results** — no results are fabricated or exaggerated. The metrics table in the
  README and notebooks are taken directly from the paper's Tables 02–05.
- **Academic conclusions** — all interpretations are grounded in the paper's findings.
